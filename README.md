# 트렌드핏 TrendFit — Phase 0 MVP

운동 기록의 추세를 읽고 다음 주를 처방하는 "의지력 대행" 앱의 핵심 루프(기록 → 추세 → 처방) 검증용 MVP.
사업 기획서의 Phase 0 과업(0-A ~ 0-E)을 웨이트 종목 기준으로 구현했다.

## 핵심 루프

1. 온보딩에서 신체 정보를 입력하고, 초보/경력자 경로를 선택한다.
   - 초보: 이번 주는 기록만 남기고 2주차부터 처방 시작.
   - 경력자: 최근 3주 3대 운동(스쿼트·벤치프레스·데드리프트) 합산 볼륨을 입력해 바로 추세 시작점을 확보.
2. 주차별로 운동(무게·횟수·세트)을 기록하면 총 볼륨(무게 × 횟수 × 세트)이 자동 계산된다.
3. 주간 총 볼륨 추세를 그래프로 확인한다.
4. "다음 주 처방 받기"를 누르면 최근 추세를 근거로 다음 주 목표 볼륨과 미션이 생성된다.

## 처방 로직 (기획서 5-1 원칙 구현)

- **볼륨 해석의 정직성**: 볼륨 상승이 세트당 평균 강도(고중량) 때문인지, 반복 수 증가(저중량 반복) 때문인지 구분한다 (`src/lib/volume.ts`의 `classifyVolumeDriver`).
- **정체 감지**: 최근 3주 볼륨 변동폭이 3% 이내면 정체 구간으로 판정하고 변화(증량)를 강제 처방한다 (`isStagnant`).
- **근거 노출**: 처방 사유에 실제 볼륨 수치를 인용한다.
- **미션형 프레이밍**: "이번 주 미션: ... (목표 N,NNNkg)" 한 문장으로 제시한다.

처방은 두 가지 경로로 생성된다.

- **AI 처방** (`src/lib/aiClient.ts`, `src/lib/prescriptionPrompt.ts`): Anthropic API 키를 등록하면 브라우저에서 직접 Messages API를 호출해 위 4원칙을 반영한 프롬프트로 처방을 생성한다. 키는 로컬 저장소에만 저장되며 서버로 전송되지 않는다.
- **규칙 기반 처방** (`src/lib/prescription.ts`의 `generateRuleBasedPrescription`): API 키 없이도 핵심 루프를 검증할 수 있도록 동일한 4원칙을 코드로 구현한 폴백 로직. 정체/고중량 성장/저중량 반복/볼륨 감소/소폭 정체/상승 흐름 6가지 상황을 구분해 처방한다.

## 기술 스택

노코드 빌더 대신 React + TypeScript + Vite로 직접 구현했다 (0-A의 실질적 대체 — 처방 프롬프트 품질이라는 핵심 자산에 집중하기 위해 화면·데이터 계층은 코드로 빠르게 확보). PWA(`vite-plugin-pwa`)로 빌드해 홈 화면 설치를 지원한다. 데이터는 1인 사용 기준으로 브라우저 로컬 저장소에 저장한다 (Phase 1의 다중 사용자 백엔드는 별도 과업).

## 실행 방법

```bash
npm install
npm run dev       # 개발 서버
npm run build     # 프로덕션 빌드 (dist/)
npm run preview   # 빌드 결과 미리보기
npm run lint      # 타입 체크
```

AI 처방을 사용하려면 앱 하단 "AI API 키 설정"에서 Anthropic API 키를 등록한다. 키가 없으면 자동으로 규칙 기반 처방으로 동작해 핵심 루프를 계속 검증할 수 있다.

## Phase 0 대응 현황

| 코드 | 과업 | 상태 |
|---|---|---|
| 0-A | 노코드 빌더 선정 | 코드 기반 구현으로 대체 (React + Vite PWA) |
| 0-B | 볼륨 입력 화면 | `WorkoutLogForm` — 무게·횟수·세트 입력 → 볼륨 자동 계산 |
| 0-C | 처방 프롬프트 v1 | `prescriptionPrompt.ts` — 4원칙 반영 |
| 0-D | AI API 연동 | `aiClient.ts` — 버튼 클릭 시 처방 표시, 키 없으면 규칙 기반 폴백 |
| 0-E | 실사용 검증 | 온보딩에서 본인 과거 볼륨을 입력해 추세·처방을 바로 확인 가능 |
